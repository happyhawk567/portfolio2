Re-designing BRTYE's "Join Us" Page:

responsive redesign

Summary: Analayzed and re-designed a web-page to improve its usability, learnability, and memorability

Process: Created a list of the page's areas of improvement & accessibility issues, then created screen-responsive Balsamiq wireframes & Figma designs to address each of the outlined issues. I then used HTML & CSS to create the re-designed page.

Reflection: I was surprised by the number of the page's issues that were not immediately apparent until I tried to use it on my phone. It's especially important when thinking about how these issues impact accessibility, such as color contrast.

Introduction

For this project, I chose to re-design one of the pages (“Join Us”) for Brown University’s BRYTE (Brown Refugee Youth Tutoring and Enrichment). I chose to focus on this website because I really admire the organization and I noticed there were a few areas of improvement in regards to the site's overall cohesiveness and responsiveness. Since the "Join Us" page is often the first stop for prospective members, I thought it would be especially important to improve this page so that it could leave a better first impression. You can access the current page here or view it below.

Issues

After spending some time interacting with the page, I have compiled a list of areas for improvement based on the standards of usability, learnability, and memorability. I also ran the page through WebAIM WAVE to detect potential accessibility issues.

Usability

  • Text gets cluttered and hard to read when screen width is small.
  • When opening the page on a phone, the navigation bar is very cluttered and hard to use. For general decreased screen width, the "donate" portion of the navigation bar overlaps with other navigation bar links.
  • There could be more contrast between text and background colors for some elements, such as the donate link on the yellow background, and the translucent white hamburger menu.

Learnability

  • The navigation bar has no visual distinguisher for the current page, and the title of the page is not written on the page.
  • There is no clear element to click in order to apply for one of the listed positions. Users have to have read through the text, where the link to the application document is found as a link within the body of text.

Memorability

  • There are a lot of different colors, which is not good for memorability and creating a cohesive page.
  • For most screen sizes, the majority of the images are obscured with text on top.
  • Logos can be important parts of a site's theme, but some parts of the BRYTE's logo on the navigation bar are hidden when screen height is decreased.
  • Longer chunks of text which could be broken up with the use of icons.

WAVE Results

WAVE results included several empty links, corresponding social media icons on the bottom of the page. This could easily be fixed by replacing the empty link text with the proper links to the organization's social media accounts. Another highlighted issue is an image's alt text, which can be added as an image property in the HTML file. Another issue presented was a low-contrast error for the white "Donate" text on top of a yellow background, which I have already listed above as a usabillity issue.

Lo-Fi Design

Using Balsamiq, I created lo-fi designs for a re-designed web-page that addresses the issues listed above. There are seperate lo-fi designs for phone, tablet, and desktop screen sizes in order to see how the page layout adapts to different screen sizes. Each of the diagrams is labeled with the corresponding number to the issue that it addresses, as detailed in the table below.
